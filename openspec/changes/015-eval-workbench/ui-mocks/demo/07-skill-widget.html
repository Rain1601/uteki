how the widget detects unverified state
1.EvolutionStore 已经在 hash prompt 文件(guardrails.md + SKILL.md + references),版本变化时 auto-bump。这是 015 之前就有的能力。
2.015 在 BenchmarkRun.metrics_summary 加 approved_by / rejected_by 字段。
3.Widget 逻辑:当前 prod skill version 是否有 BenchmarkRun 标 approved_by != null?
→ 有 → 显示 verified state
→ 没有 → 显示 unverified state(+ 显示跑 bench 按钮)
→ 当前在跑 → 显示 running state
4."Skip" 按钮:写一个特殊的 BenchmarkRun row(mode=skip),记录"用户明确表示不验证"。Widget 不再提醒。可后续撤销。
